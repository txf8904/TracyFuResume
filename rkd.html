RKD Game

Game Review APP | UX/UI Designer | Course Project | Figma

Project Overview

RKD Game is a game review app that provides a series of reviews about the classic and newest arcade games and provides the gaming community for gamers. The app is geared toward people who have an interest in arcade games and looking for friends to play together with.

Understanding the user

Problem:

Arcade gamers often can't find people who have similar interests in the real life. Unlike online games, they are able to community with each other through voice chat that builds into the game. The arcade games are more like offline, face-to-face games. Therefore, arcade gamers are looking for a platform that can provide information and news about arcade games and find their own unique gamer community.

Solution:

The application provides people with information about classic and newest arcade games, so no matter whether the user is a new arcade gamer or a fanatic arcade gamer, you can find your game here. Moreover, there will never be alone because there is always a community for you to share your fun with each other.

Persona:

Lewis is a 3rd-year college student who needs updated on the newest game every week because he has a heavy schedule for his college.

The Design

Wireframes:

For the home screen, I realized that quick access to interesting games process helps users save time. Meanwhile, the users also need to explore the newest/popular games.

Moreover,the wireframes include the ranking page, search/menu page, discover page, and game community/group page.

Based on the digital wireframes, the low-fidelity prototype has a primary user flow that finds a game review from different entry points in a quick and easy way.

Refining the design

Usability Study:

Since we are creating a new app, we need to find out if the main user experience, finding a game review, and connecting to a game community, are easy for users to complete.

The usability study focuses on five questions:
      1.How long does it take a user to find a game review?
      2. What can we learn from the user flow to find a game review?
      3. Are there parts of the user flow where users get stuck?
      4. Are there more features that users would like to see included in the app?
      5. Do users think the app is easy or difficult?

To solve these questions, I conducted an unmoderated usability study that has five participants involved with different gender and age gap. After watching and talking with participants, I collected their options and summarized them into three insights.

     Insights:
      1. Search: Users need a way to find a game review more quickly without having to go through the Ranking or Discover
      2. Interface: Users need a more friendly interface to move smoothly in the app
      3. Game Community: Users need more connections with friends and others to have more fun outside of the game

Based on the insights, I made following changes:

First, I added a quick search feature for the search function. It is not only allowed users to find the hottest topic but also can help users to limit the result to what they are looking for.

Also, the early design allowed users to use the search functions in a specific location. I added the search bar at the top of the screen, so the users can access it anytime. Moreover, change the navigation system from a pop-up menu to a fixed position. Last, I refine the game community more clearly and add login/sign-up pages.

Final

Summary:

RKD Game App is my first UX design project in my study journey. While designing the RKD Game App, I started to understand the process of a full UX design cycle. Along the way, I learned that the initial ideas for the app are just a start point. The final product is not what I as a designer want, it is about what users want and need.

At the beginning of the design, I assumed that I was a user who doesn't really into arcade games but wants to find some new game. Therefore, as a new gamer, I assumed that users want to the app tell them what to play instead of helping them find what are they looking for. Moreover, my initial research was off-topic a little bit, it is more focus on what kind of game reviews users want instead of what kind of features they are looking for in a game review app.

As a result, the first usability study shows that a lot of participants can't find game reviews as they want because I ignored the importance of search. In addition, I also learned that usability studies are the key element in the process through the design.

It is an exciting and confusing process, but I eventually get over the difficulty and call it a success.
